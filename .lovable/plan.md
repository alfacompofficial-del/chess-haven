# Шахматы онлайн — план реализации

Масштабный проект. Разобью на этапы, чтобы каждый шаг был проверяемым. Подтвердите план, и начну с этапа 1.

## Технологический стек

- TanStack Start + React 19 + Tailwind v4 (уже есть)
- Lovable Cloud (Supabase) — БД, auth, realtime для партий/уведомлений
- Lovable AI Gateway → Gemini 2.5 Flash для ИИ-соперника
- chess.js — правила, валидация ходов, мат/пат/повторение
- react-chessboard — визуализация доски
- i18next — переводы интерфейса
- Определение страны по IP — ipapi.co (бесплатно, без ключа)

## Этап 1. Фундамент

- Подключить Lovable Cloud
- Схема БД:
  - `profiles` (id→auth.users, nickname unique, avatar_url, country_code, rating_rapid=200, rating_blitz=200, rating_bullet=200, language, board_theme, premoves_enabled)
  - `friendships` (user_id, friend_id, status: pending/accepted)
  - `games` (white_id, black_id, time_control, mode: online/ai/tournament, pgn, fen, status, winner, result_type)
  - `tournaments` (creator_id, name, duration_minutes, status, started_at)
  - `tournament_participants`, `tournament_matches`
  - `notifications` (user_id, type, payload, read)
- RLS + GRANT для всех таблиц
- Таблица ролей `user_roles` + `has_role()`
- Аутентификация: ник + пароль (используем `<nickname>@chess.local` внутри auth.users, ник показывается из profiles; ник неизменяем после регистрации)
- Триггер auto-create profile при регистрации: определение страны по IP через edge-подобную server function

## Этап 2. Дизайн-система и layout

- Тёмная шахматная тема (не фиолетовая), деревянно-угольная палитра, семантические токены в `styles.css`
- Главный layout:
  - Верх-право: кнопки «Играть», «Турниры», «Настройки», «Топ лидеров»
  - Низ-право: круглый аватар профиля
  - Над аватаром: колокольчик уведомлений с бейджем
  - Поисковик игроков сверху (для запросов в друзья)

## Этап 3. Профиль

- Модалка при клике на круг: смена аватарки (Supabase Storage), смена отображаемого имени (не ника!)
- Отображение флага + страны, рейтингов Рапид/Блиц/Пуля

## Этап 4. Друзья и уведомления

- Поиск игроков по нику
- Отправка/приём/отклонение запросов
- Realtime-обновление колокольчика

## Этап 5. Игра

- Модалка «Играть»: выбор режима (Друг онлайн / ИИ), выбор контроля времени (Пуля 1+0/2+1, Блиц 3+0/5+0, Рапид 10+0/15+10)
- Онлайн с другом: приглашение → realtime-канал партии, синхронизация ходов, часы, премувы (если включены)
- ИИ (Gemini 2.5 Flash): server function получает FEN → просит модель вернуть ход в UCI, валидирует через chess.js, при невалидном — повтор. Против ИИ **нет** кнопок «сдаться»/«ничья»
- Оверлеи игрока: аватар, ник, рейтинг, флаг (клик → название страны на 2 сек через tooltip)
- Правила: мат/пат/троекратное повторение автоматически; рокировка по всем условиям (chess.js уже это делает)
- Рейтинг: +20 победа, 0 ничья, −10 поражение, не ниже 0. Обновление через server function (только после завершённой партии)

## Этап 6. Турниры

- Список турниров, кнопка «Создать» (название + длительность в минутах)
- Вход в турнир: таблица лидеров, обратный отсчёт
- Матчмейкинг: свободных участников автоматически сводит парами (Swiss-подобный простой алгоритм)

## Этап 7. Настройки

- Язык интерфейса (RU/EN как минимум)
- Тема доски (классика/дерево/неон/зелёная)
- Разрешить премувы (on/off)
- Дополнительно: звуки ходов, показ подсказок легальных ходов, авто-проматывание пешки в ферзя

## Этап 8. Топ лидеров

- Три вкладки: Рапид / Блиц / Пуля
- Топ-100 по рейтингу с флагами

## Технические детали

- Все ходы валидируются на клиенте и на сервере (server function записывает партию)
- Realtime через Supabase channels для партий/уведомлений/турниров
- Gemini через `createServerFn` + Lovable AI Gateway, модель `google/gemini-2.5-flash`
- SEO: title/description на каждом маршруте, sitemap.xml
